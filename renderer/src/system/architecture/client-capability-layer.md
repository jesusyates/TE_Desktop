# Client Capability Layer（端能力层）

> 原则：**三端共享 Shared Core Backend**（账号、支付、任务、结果、模板、记忆、安全策略、能力注册等）；差异仅在**各端能做什么、如何触达本地资源**，不在「再造一套核心业务」。

---

## 三端定位

| 端 | 形态 | 与核心的关系 |
|----|------|----------------|
| **Desktop** | 主工作台、深度执行 | 全量或高权重消费核心服务；**本地强能力**经 Local Execution Runtime，策略仍受核心约束 |
| **Web** | 辅助：账户、订阅、轻任务 | 消费同一套核心 API；**不承担**桌面级本地执行 |
| **Mobile** | 随身入口、轻量与确认 | 消费同一套核心 API；**侧重发起、查看、通知与轻内容** |

---

## Desktop

| 能力域 | 边界说明 |
|--------|----------|
| **文件系统** | 在用户设备上读写、整理、监听约定目录（经运行时与权限模型） |
| **软件控制** | 启动、窗口、自动化通道等与 OS 应用的交互 |
| **本地 agent** | 设备侧编排与扩展点（仍受核心策略与能力注册约束） |
| **本地事件流** | 执行过程可观测、可回放的端上事件管线 |
| **高权限操作** | 仅在明确许可与策略允许下执行；**不是**独立于核心的第二套计费/账号体系 |

---

## Web

| 能力域 | 边界说明 |
|--------|----------|
| **账号** | 登录、绑定、设置；数据归属 User/Auth 核心服务 |
| **支付** | 收银台、订阅管理入口；计费事实在 Billing 核心 |
| **历史** | 任务/结果列表与详情只读或轻量操作；权威数据在 Task/Result 核心 |
| **模板** | 选用、编辑轻量参数；权威在 Template 核心 |
| **内容任务** | 无本地强执行时的生成类任务 |
| **轻量执行** | 不替代 Desktop 本地运行时 |

---

## Mobile

| 能力域 | 边界说明 |
|--------|----------|
| **发起任务** | 创建任务、附加上下文；与 Task 核心一致 |
| **查看历史** | 与 Web/Desktop 同源列表与状态 |
| **通知** | 承接 Notification 核心的触达 |
| **远程确认** | 高风险或权限类人工确认的消费端之一 |
| **轻量内容任务** | 短文、摘要类；**不**承担桌面级自动化 |

---

## 小结

- **不得**在 Desktop / Web / Mobile **分别实现**独立的支付、账号、模板、记忆或结果权威存储。
- **端能力层**描述的是**能做什么**；**谁保存真相**一律在 **Shared Core Backend**。
