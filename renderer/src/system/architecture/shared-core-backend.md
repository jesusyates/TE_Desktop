# Shared Core Backend（统一核心后端）

> 职责边界：**唯一核心业务后端**。各端只消费能力，不复制账号、计费、任务、结果、模板、记忆与安全策略。

---

## Auth Service

| 维度 | 说明 |
|------|------|
| **作用** | 身份认证、会话与令牌生命周期、设备与会话绑定策略入口 |
| **负责的数据** | 用户身份凭据关系、会话/刷新令牌元数据、登录态审计引用（细目由 Audit 协同） |
| **哪些端会使用** | Desktop、Web、Mobile |

## User Service

| 维度 | 说明 |
|------|------|
| **作用** | 用户档案、偏好引用、账户级设置与可见性边界 |
| **负责的数据** | 用户基础档案、与订阅/权益关联外键、跨产品用户标识 |
| **哪些端会使用** | Desktop、Web、Mobile |

## Billing Service

| 维度 | 说明 |
|------|------|
| **作用** | 订阅、订单、发票与权益（Entitlements）的统一判定入口 |
| **负责的数据** | 订阅状态、权益快照、订单与支付关联（具体渠道由 Market 配置约束） |
| **哪些端会使用** | Desktop、Web、Mobile（Mobile 以只读/跳转为主时可轻量） |

## Usage Metering Service

| 维度 | 说明 |
|------|------|
| **作用** | 用量采集、聚合、配额与限流依据 |
| **负责的数据** | 调用计数、资源消耗事件、配额策略执行所需计量事实 |
| **哪些端会使用** | Desktop、Web、Mobile（上报与查询经统一 API） |

## Task Service

| 维度 | 说明 |
|------|------|
| **作用** | 任务创建、状态机、步骤/Run 与执行会话在核心的权威记录 |
| **负责的数据** | 任务实体、运行记录、与结果/记忆/审计的关联 |
| **哪些端会使用** | Desktop、Web、Mobile |

## Result Service

| 维度 | 说明 |
|------|------|
| **作用** | 任务结果包的持久化、版本与检索；与模板/回放消费的统一源 |
| **负责的数据** | 结构化结果快照、导出记录引用、与 Task 的绑定 |
| **哪些端会使用** | Desktop、Web、Mobile |

## Template Service

| 维度 | 说明 |
|------|------|
| **作用** | 模板库、版本、共享范围与市场可见性（结合 Country Layer） |
| **负责的数据** | 模板定义、变量 Schema、来源任务追溯字段 |
| **哪些端会使用** | Desktop、Web、Mobile |

## Memory Service

| 维度 | 说明 |
|------|------|
| **作用** | 长期记忆、偏好与上下文的权威存储与检索（产品内「越用越懂」） |
| **负责的数据** | 记忆条目、嵌入/索引元数据、用户与任务维度的记忆分区 |
| **哪些端会使用** | Desktop、Web、Mobile |

## Safety Policy Service

| 维度 | 说明 |
|------|------|
| **作用** | 安全与合规策略的判定、策略版本与下发；**端侧拦截须与此对齐** |
| **负责的数据** | 策略规则集、命中与阻断原因码、与 Audit 的关联 |
| **哪些端会使用** | Desktop、Web、Mobile（服务端权威；端侧为执行前闸） |

## Capability Registry Service

| 维度 | 说明 |
|------|------|
| **作用** | 能力目录、上架与审核状态、端类型与权限前置条件声明的权威源 |
| **负责的数据** | 能力 ID、元数据、审核生命周期、与本地运行时的契约引用（非本地实现） |
| **哪些端会使用** | Desktop、Web、Mobile |

## Market / Locale Config Service

| 维度 | 说明 |
|------|------|
| **作用** | 市场、语言、功能开关与跨境规则的**配置下发**入口（常与 Country Layer 数据同源） |
| **负责的数据** | market/locale 键空间下的功能旗标、默认可用能力列表、文案/模板包指针 |
| **哪些端会使用** | Desktop、Web、Mobile |

## Audit / Risk Service

| 维度 | 说明 |
|------|------|
| **作用** | 审计日志、风险事件、调查与处置状态 |
| **负责的数据** | 不可篡改审计记录、风险评分事件、关联 task/user/session |
| **哪些端会使用** | 主要消费在运营/后台与合规链路；各端仅产生经 SDK 的受控上报 |

## Notification Service

| 维度 | 说明 |
|------|------|
| **作用** | 跨端通知编排：触达渠道选择、模板渲染、投递状态 |
| **负责的数据** | 通知意图、投递记录、用户通知偏好引用 |
| **哪些端会使用** | Desktop、Web、Mobile（各端为展示与本地推送承接，规则在核心） |
