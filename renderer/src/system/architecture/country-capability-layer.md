# Country Capability Layer（国家 / 地区能力层）

> 定位：**配置与策略层**，不是前端页面层。市场差异通过数据与策略驱动，**禁止**散落成 Web/Desktop/Mobile 各自硬编码。

---

## 职责概览

| 主题 | 说明 |
|------|------|
| **market / locale** | 用户所属市场、语言与区域默认；作为配置查询主键 |
| **国家能力开关** | 某市场在核心侧是否开放某类能力、功能旗标 |
| **国家支付方式** | 与 Billing 配合的可用渠道、货币与结算规则入口 |
| **国家定价** | 套餐、单价、税额规则的配置层（价格在核心结算中生效） |
| **国家安全策略差异** | Safety Policy 在规则强度、必选确认、禁区列表上的市场差异 |
| **国家合规限制** | 数据驻留、日志留存、禁止场景等合规模块开关 |
| **国家模板 / 内容差异** | 默认模板包、本地化内容集指针、市场推荐模板与素材引用；**权威仍在 Template 核心**，非各端私自复制一套 |

---

## 与上下层关系

| 方向 | 关系 |
|------|------|
| **向上（产品策略）** |  product 只消费「某 market 下effective 配置」，不直接写死国别 if-else |
| **向下（Shared Core）** | Market / Locale Config、Safety、Billing 等**读取**本层或与本层数据模型对齐 |
| **向旁（Client）** | 客户端只携带 **market + client platform** 等头/上下文；**不实现**国别业务真理源 |

---

## 原则

- 国别差异 = **可版本化的配置 + 策略**，不是复制多套前端或多套核心库。
- 前端与业务编排代码 **不得** 成为市场规则的堆放处。
